## Overview

The Teams API provides endpoints for managing teams, team membership, and team-related operations in Mattermost.

## Create Team

Create a new team on the system.

```http
POST /api/v4/teams
```

### Request Body

<ParamField body="name" type="string" required>
  Unique handler for the team (will be in the team URL)
</ParamField>

<ParamField body="display_name" type="string" required>
  Non-unique display name for the team
</ParamField>

<ParamField body="type" type="string" required>
  Team type: `O` for open, `I` for invite-only
</ParamField>

### Example Request

<CodeGroup>
```bash cURL
curl -X POST http://localhost:8065/api/v4/teams \
  -H 'Authorization: Bearer your-token' \
  -H 'Content-Type: application/json' \
  -d '{
    "name": "engineering",
    "display_name": "Engineering Team",
    "type": "O"
  }'
```

```javascript JavaScript
const response = await fetch('http://localhost:8065/api/v4/teams', {
  method: 'POST',
  headers: {
    'Authorization': 'Bearer your-token',
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    name: 'engineering',
    display_name: 'Engineering Team',
    type: 'O'
  })
});
```
</CodeGroup>

### Response

<ResponseField name="id" type="string">
  Unique team identifier
</ResponseField>

<ResponseField name="name" type="string">
  Team URL handle
</ResponseField>

<ResponseField name="display_name" type="string">
  Team display name
</ResponseField>

<ResponseField name="type" type="string">
  Team type (O=open, I=invite-only)
</ResponseField>

<ResponseField name="create_at" type="integer">
  Creation timestamp in milliseconds
</ResponseField>

```json Response Example
{
  "id": "team123abc",
  "create_at": 1507840900004,
  "update_at": 1507840900004,
  "delete_at": 0,
  "display_name": "Engineering Team",
  "name": "engineering",
  "description": "",
  "email": "",
  "type": "O",
  "allow_open_invite": true
}
```

## Get Team

Retrieve a single team by ID.

```http
GET /api/v4/teams/{team_id}
```

### Path Parameters

<ParamField path="team_id" type="string" required>
  Team GUID
</ParamField>

### Example Request

```bash
curl -H 'Authorization: Bearer your-token' \
  http://localhost:8065/api/v4/teams/{team_id}
```

## Get Teams

Retrieve a paginated list of teams.

```http
GET /api/v4/teams
```

### Query Parameters

<ParamField query="page" type="integer" default="0">
  Page number to retrieve
</ParamField>

<ParamField query="per_page" type="integer" default="60">
  Number of teams per page
</ParamField>

<ParamField query="include_total_count" type="boolean" default="false">
  Include total count in response
</ParamField>

<ParamField query="exclude_policy_constrained" type="boolean" default="false">
  Exclude teams with data retention policies (requires `sysconsole_read_compliance`)
</ParamField>

### Example Request

```bash
curl -H 'Authorization: Bearer your-token' \
  'http://localhost:8065/api/v4/teams?page=0&per_page=50'
```

## Update Team

Update a team's information.

```http
PUT /api/v4/teams/{team_id}
```

### Request Body

<ParamField body="id" type="string" required>
  Team ID (must match path parameter)
</ParamField>

<ParamField body="display_name" type="string" required>
  Team display name
</ParamField>

<ParamField body="description" type="string" required>
  Team description
</ParamField>

<ParamField body="company_name" type="string" required>
  Company name
</ParamField>

<ParamField body="allowed_domains" type="string" required>
  Comma-separated list of allowed email domains
</ParamField>

<ParamField body="invite_id" type="string" required>
  Team invite ID
</ParamField>

<ParamField body="allow_open_invite" type="boolean" required>
  Allow open invitations
</ParamField>

### Example Request

```bash
curl -X PUT http://localhost:8065/api/v4/teams/{team_id} \
  -H 'Authorization: Bearer your-token' \
  -H 'Content-Type: application/json' \
  -d '{
    "id": "team123abc",
    "display_name": "Engineering Team",
    "description": "Our engineering department",
    "company_name": "Acme Corp",
    "allowed_domains": "acme.com",
    "invite_id": "inviteid123",
    "allow_open_invite": true
  }'
```

## Delete Team

Soft delete a team (marks as deleted in database).

```http
DELETE /api/v4/teams/{team_id}
```

### Query Parameters

<ParamField query="permanent" type="boolean" default="false">
  Permanently delete the team (requires `ServiceSettings.EnableAPITeamDeletion = true`)
</ParamField>

### Example Request

```bash
curl -X DELETE \
  -H 'Authorization: Bearer your-token' \
  http://localhost:8065/api/v4/teams/{team_id}
```

<Warning>
Permanent deletion cannot be undone and removes all team data.
</Warning>

<ParamField body="per_page" type="string">

</ParamField>

</ParamField>

<ParamField body="group_constrained" type="boolean">
  Filter by group_constrained setting
</ParamField>

### Example Request

```bash

  -H 'Authorization: Bearer your-token' \
  -H 'Content-Type: application/json' \
  -d '{"term": "engineering"}'
```

## Invite Users to Team

Invite users to a team via email.

```http
POST /api/v4/teams/{team_id}/invite/email
```

### Request Body

Array of email addresses:

```json
["user1@example.com", "user2@example.com"]
```

<Note>
Rate limited to 20 emails per hour with a burst of 20.
</Note>

### Example Request

```bash
curl -X POST http://localhost:8065/api/v4/teams/{team_id}/invite/email \
  -H 'Authorization: Bearer your-token' \
  -H 'Content-Type: application/json' \
  -d '["john@example.com", "jane@example.com"]'
```


```bash

  -H 'Authorization: Bearer your-token' \
```


## Permissions

| Operation | Required Permission |
|-----------|--------------------|
| Create team | `create_team` |
| Get team | `view_team` |
| Update team | `manage_team` |
| Delete team | `manage_team` |
| Add member | `add_user_to_team` or self on open team |
| Remove member | `remove_user_from_team` or self |
| Update roles | `manage_team_roles` |
| Invite users | `invite_user` and `add_user_to_team` |

## Error Codes

| Code | Description |
|------|-------------|
| `400` | Invalid parameters |
| `401` | Not authenticated |
| `403` | Insufficient permissions |
| `404` | Team not found |
