# Installation Guide

This guide covers production-ready deployments of Mattermost across different platforms and environments. For a quick development setup, see the [Quickstart Guide](/quickstart).

<Note>
  Mattermost is written in **Go and React**, runs as a **single Linux binary**, and requires **PostgreSQL** for data storage.
</Note>

## System Requirements

Before installing Mattermost, ensure your system meets these minimum requirements:

### Hardware Requirements

<Tabs>
  <Tab title="Small Team (1-1,000 users)">
    - **CPU**: 2 vCPUs/cores
    - **RAM**: 4GB
    - **Storage**: 10GB (plus storage for files and attachments)
    - **Database**: 1 vCPU, 2GB RAM
  </Tab>
  <Tab title="Medium Team (1,000-10,000 users)">
    - **CPU**: 4 vCPUs/cores
    - **RAM**: 8GB
    - **Storage**: 20GB (plus storage for files and attachments)
    - **Database**: 2 vCPUs, 4GB RAM
  </Tab>
  <Tab title="Large Team (10,000+ users)">
    - **CPU**: 8+ vCPUs/cores (clustered deployment)
    - **RAM**: 16GB+ per node
    - **Storage**: 50GB+ (plus dedicated file storage)
    - **Database**: 4+ vCPUs, 8GB+ RAM (with replication)
  </Tab>
</Tabs>

### Software Requirements

- **Operating System**: Ubuntu 20.04+, RHEL 8+, Debian 10+, or compatible Linux distribution
- **Database**: PostgreSQL 14+ (recommended), or PostgreSQL 12+ (minimum)
- **Go Runtime**: Go 1.24.13 (for building from source)
- **Node.js**: Version specified in `.nvmrc` (for webapp development)
- **Reverse Proxy**: NGINX, Apache, or HAProxy (recommended for SSL termination)

<Warning>
  MySQL support has been deprecated. PostgreSQL is the only supported database for new installations.
</Warning>

## Method 1: Docker Deployment

Deploy Mattermost using Docker for a containerized, production-ready setup.

### Prerequisites

- Docker Engine 20.10+
- Docker Compose 2.0+
- Domain name with DNS configured (for SSL)

### Step 1: Prepare the Environment

```bash
# Create installation directory
mkdir -p /opt/mattermost
cd /opt/mattermost

# Create directory structure
mkdir -p {config,data,logs,plugins}
```

### Step 2: Create Production Docker Compose File

Create `docker-compose.yml`:

```yaml docker-compose.yml
services:
  postgres:
    image: postgres:14
    container_name: mattermost-postgres
    restart: unless-stopped
    security_opt:
      - no-new-privileges:true
    pids_limit: 100
    read_only: true
    tmpfs:
      - /tmp
      - /var/run/postgresql
    environment:
      # Database credentials - CHANGE THESE!
      POSTGRES_USER: mmuser
      POSTGRES_PASSWORD: ${POSTGRES_PASSWORD}
      POSTGRES_DB: mattermost
      # PostgreSQL configuration for Mattermost
      POSTGRES_INITDB_ARGS: "--auth-host=scram-sha-256 --auth-local=scram-sha-256"
    volumes:
      - postgres_data:/var/lib/postgresql/data
    healthcheck:
      test: ["CMD", "pg_isready", "-h", "localhost", "-U", "mmuser"]
      interval: 10s
      timeout: 5s
      retries: 5
    networks:
      - mattermost

  mattermost:
    image: mattermost/mattermost-enterprise-edition:latest
    container_name: mattermost
    restart: unless-stopped
    security_opt:
      - no-new-privileges:true
    pids_limit: 200
    depends_on:
      postgres:
        condition: service_healthy
    environment:
      # Timezone
      TZ: UTC
      
      # Database Configuration
      MM_SQLSETTINGS_DRIVERNAME: postgres
      MM_SQLSETTINGS_DATASOURCE: postgres://mmuser:${POSTGRES_PASSWORD}@postgres:5432/mattermost?sslmode=disable&connect_timeout=10
      
      # Server Settings
      MM_SERVICESETTINGS_SITEURL: https://mattermost.example.com
      MM_SERVICESETTINGS_LISTENADDRESS: :8065
      MM_SERVICESETTINGS_ENABLELOCALMODE: true
      
      # File Storage (use S3 for production)
      MM_FILESETTINGS_DRIVERNAME: local
      MM_FILESETTINGS_DIRECTORY: /mattermost/data
      
      # Performance Settings
      MM_SQLSETTINGS_MAXIDLECONNS: 20
      MM_SQLSETTINGS_MAXOPENCONNS: 300
      MM_SQLSETTINGS_QUERYTIMEOUT: 30
      
      # Cluster Settings (for High Availability)
      MM_CLUSTERSETTINGS_ENABLE: false
      MM_CLUSTERSETTINGS_CLUSTERNAME: production
      
      # Plugin Settings
      MM_PLUGINSETTINGS_ENABLE: true
      MM_PLUGINSETTINGS_ENABLEUPLOADS: true
    ports:
      - "8065:8065"
      - "8067:8067"  # Metrics
      - "8074:8074"  # Cluster gossip
      - "8075:8075"  # Performance monitoring
    volumes:
      - ./config:/mattermost/config:rw
      - ./data:/mattermost/data:rw
      - ./logs:/mattermost/logs:rw
      - ./plugins:/mattermost/plugins:rw
      - ./client/plugins:/mattermost/client/plugins:rw
    healthcheck:
      test: [
        "CMD",
        "/mattermost/bin/mmctl",
        "system",
        "status",
        "--local"
      ]
      interval: 30s
      timeout: 10s
      retries: 3
      start_period: 90s
    networks:
      - mattermost

volumes:
  postgres_data:
    driver: local

networks:
  mattermost:
    driver: bridge
```

<Note>
  This configuration is based on the production Dockerfile found in `server/build/Dockerfile` and the docker-compose setup in `server/build/docker-compose.common.yml`.
</Note>

### Step 3: Configure Environment Variables

Create `.env` file for sensitive configuration:

```bash .env
# PostgreSQL Password
POSTGRES_PASSWORD=your_secure_password_here

# Mattermost Domain
MATTERMOST_DOMAIN=mattermost.example.com
```

<Warning>
  **Security**: Use strong, unique passwords. Never commit `.env` to version control.
</Warning>

### Step 4: Start Mattermost

```bash
# Start services
docker compose up -d

# Verify services are running
docker compose ps

# Check logs
docker compose logs -f mattermost
```

### Step 5: Configure SSL/TLS (Production Required)

For production deployments, configure NGINX as a reverse proxy:

```nginx /etc/nginx/sites-available/mattermost
upstream mattermost_backend {
    server localhost:8065;
    keepalive 32;
}

proxy_cache_path /var/cache/nginx levels=1:2 keys_zone=mattermost_cache:10m max_size=3g inactive=120m use_temp_path=off;

server {
    listen 80;
    server_name mattermost.example.com;
    return 301 https://$server_name$request_uri;
}

server {
    listen 443 ssl http2;
    server_name mattermost.example.com;

    ssl_certificate /etc/letsencrypt/live/mattermost.example.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/mattermost.example.com/privkey.pem;
    ssl_session_timeout 1d;
    ssl_protocols TLSv1.2 TLSv1.3;
    ssl_ciphers 'ECDHE-ECDSA-AES128-GCM-SHA256:ECDHE-RSA-AES128-GCM-SHA256:ECDHE-ECDSA-AES256-GCM-SHA384:ECDHE-RSA-AES256-GCM-SHA384';
    ssl_prefer_server_ciphers off;

    location ~ /api/v[0-9]+/(users/)?websocket$ {
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        client_max_body_size 50M;
        proxy_set_header Host $http_host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_set_header X-Frame-Options SAMEORIGIN;
        proxy_buffers 256 16k;
        proxy_buffer_size 16k;
        client_body_timeout 60;
        send_timeout 300;
        lingering_timeout 5;
        proxy_connect_timeout 90;
        proxy_send_timeout 300;
        proxy_read_timeout 90s;
        proxy_pass http://mattermost_backend;
    }

    location / {
        client_max_body_size 50M;
        proxy_set_header Connection "";
        proxy_set_header Host $http_host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_set_header X-Frame-Options SAMEORIGIN;
        proxy_buffers 256 16k;
        proxy_buffer_size 16k;
        proxy_read_timeout 600s;
        proxy_cache mattermost_cache;
        proxy_cache_revalidate on;
        proxy_cache_min_uses 2;
        proxy_cache_use_stale timeout;
        proxy_cache_lock on;
        proxy_http_version 1.1;
        proxy_pass http://mattermost_backend;
    }
}
```

Enable the configuration:

```bash
# Enable site
sudo ln -s /etc/nginx/sites-available/mattermost /etc/nginx/sites-enabled/

# Test configuration
sudo nginx -t

# Reload NGINX
sudo systemctl reload nginx
```

## Method 2: Kubernetes Deployment

Deploy Mattermost on Kubernetes for cloud-native, scalable infrastructure.

### Using Helm (Recommended)

```bash
# Add Mattermost Helm repository
helm repo add mattermost https://helm.mattermost.com
helm repo update

# Create namespace
kubectl create namespace mattermost

# Install Mattermost with Operator
helm install mattermost-operator mattermost/mattermost-operator \
  --namespace mattermost

# Create Mattermost instance
kubectl apply -f - <<EOF
apiVersion: mattermost.com/v1alpha1
kind: Mattermost
metadata:
  name: mattermost-prod
  namespace: mattermost
spec:
  size: 5000users
  ingress:
    enabled: true
    host: mattermost.example.com
    annotations:
      cert-manager.io/cluster-issuer: letsencrypt-prod
  database:
    external:
      secret: mattermost-postgres-secret
  fileStore:
    external:
      url: s3.amazonaws.com
      bucket: mattermost-files
      secret: mattermost-s3-secret
EOF
```

<Tip>
  For detailed Kubernetes deployment, see the [official Kubernetes installation guide](https://docs.mattermost.com/install/install-kubernetes.html).
</Tip>

## Method 3: Ubuntu Installation

Install Mattermost natively on Ubuntu 20.04 LTS or newer.

### Step 1: Install PostgreSQL

```bash
# Update package list
sudo apt update

# Install PostgreSQL
sudo apt install -y postgresql postgresql-contrib

# Start and enable PostgreSQL
sudo systemctl start postgresql
sudo systemctl enable postgresql
```

### Step 2: Configure Database

```bash
# Create Mattermost database and user
sudo -u postgres psql <<EOF
CREATE DATABASE mattermost;
CREATE USER mmuser WITH PASSWORD 'secure_password';
GRANT ALL PRIVILEGES ON DATABASE mattermost TO mmuser;
\c mattermost
GRANT USAGE, CREATE ON SCHEMA PUBLIC TO mmuser;
\q
EOF
```

### Step 3: Download and Install Mattermost

```bash
# Download latest Mattermost Enterprise Edition
cd /tmp
wget https://latest.mattermost.com/mattermost-enterprise-linux
tar -xvzf mattermost*.gz

# Move to installation directory
sudo mv mattermost /opt/

# Create storage directory
sudo mkdir /opt/mattermost/data

# Create mattermost user
sudo useradd --system --user-group mattermost

# Set permissions
sudo chown -R mattermost:mattermost /opt/mattermost
sudo chmod -R g+w /opt/mattermost
```

<Note>
  The Mattermost binary is compiled from Go source code (version 1.24.13) and runs as a single executable.
</Note>

### Step 4: Configure Mattermost

Edit `/opt/mattermost/config/config.json`:

```json
{
  "SqlSettings": {
    "DriverName": "postgres",
    "DataSource": "postgres://mmuser:secure_password@localhost:5432/mattermost?sslmode=disable&connect_timeout=10"
  },
  "ServiceSettings": {
    "SiteURL": "https://mattermost.example.com",
    "ListenAddress": ":8065",
    "EnableLocalMode": true
  },
  "FileSettings": {
    "DriverName": "local",
    "Directory": "/opt/mattermost/data/"
  }
}
```

<Warning>
  The connection string format for PostgreSQL is:
  ```
  postgres://username:password@host:port/database?sslmode=disable&connect_timeout=10
  ```
</Warning>

### Step 5: Create Systemd Service

Create `/lib/systemd/system/mattermost.service`:

```ini
[Unit]
Description=Mattermost
After=network.target postgresql.service

[Service]
Type=notify
ExecStart=/opt/mattermost/bin/mattermost
TimeoutStartSec=3600
KillMode=mixed
Restart=always
RestartSec=10
WorkingDirectory=/opt/mattermost
User=mattermost
Group=mattermost
LimitNOFILE=49152

[Install]
WantedBy=multi-user.target
```

### Step 6: Start Mattermost

```bash
# Reload systemd
sudo systemctl daemon-reload

# Start Mattermost
sudo systemctl start mattermost

# Enable auto-start on boot
sudo systemctl enable mattermost

# Check status
sudo systemctl status mattermost
```

## Database Configuration

### PostgreSQL Optimization for Mattermost

Edit `/etc/postgresql/14/main/postgresql.conf`:

```conf
# Connection Settings
max_connections = 300
shared_buffers = 2GB
effective_cache_size = 6GB

# Query Performance
work_mem = 16MB
maintenance_work_mem = 512MB

# Write Performance
wal_buffers = 16MB
checkpoint_completion_target = 0.9

# Monitoring
log_min_duration_statement = 1000  # Log queries > 1 second
```

Restart PostgreSQL:

```bash
sudo systemctl restart postgresql
```

### Database Connection String Format

Based on the source code (`server/docker-compose.yaml:120`):

```
postgres://mmuser:password@host:5432/mattermost?sslmode=disable&connect_timeout=10
```

**Parameters:**
- `mmuser`: Database username
- `password`: Database password
- `host`: Database hostname/IP (use `localhost` or `postgres` for Docker)
- `5432`: PostgreSQL port
- `mattermost`: Database name
- `sslmode=disable`: For local/docker connections (use `require` for production)
- `connect_timeout=10`: Connection timeout in seconds

## High Availability Setup

For enterprise deployments requiring 99.9%+ uptime.

### Cluster Configuration

Enable clustering in `config.json` or via environment variables:

```bash
# Enable clustering
MM_CLUSTERSETTINGS_ENABLE=true
MM_CLUSTERSETTINGS_CLUSTERNAME=production
MM_CLUSTERSETTINGS_USEIPADDRESS=true
MM_CLUSTERSETTINGS_GOSSIPPORT=8074
MM_CLUSTERSETTINGS_STREAMINGPORT=8075
```

### Load Balancer Configuration

Use HAProxy or NGINX to distribute traffic across multiple Mattermost nodes:

```conf haproxy.cfg
frontend mattermost_frontend
    bind *:443 ssl crt /etc/ssl/mattermost.pem
    mode http
    option forwardfor
    default_backend mattermost_backend

backend mattermost_backend
    mode http
    balance roundrobin
    option httpchk GET /api/v4/system/ping
    server mattermost1 10.0.1.10:8065 check
    server mattermost2 10.0.1.11:8065 check
    server mattermost3 10.0.1.12:8065 check
```

## Post-Installation

### Initial System Console Setup

1. **Access System Console**: Navigate to `https://your-domain.com`
2. **Create Admin Account**: Set up the first system administrator
3. **Configure Site Settings**:
   - Site URL
   - Email notifications (SMTP)
   - File storage (S3/MinIO recommended for production)
4. **Enable Plugins**: Install from the Plugin Marketplace
5. **Configure Authentication**: LDAP, SAML, or OAuth

### Essential Configuration

<AccordionGroup>
  <Accordion title="Email Configuration (SMTP)">
    **System Console** → **Environment** → **SMTP**
    
    ```bash
    MM_EMAILSETTINGS_ENABLESMTPAUTH=true
    MM_EMAILSETTINGS_SMTPSERVER=smtp.gmail.com
    MM_EMAILSETTINGS_SMTPPORT=587
    MM_EMAILSETTINGS_SMTPUSERNAME=your-email@gmail.com
    MM_EMAILSETTINGS_SMTPPASSWORD=your-app-password
    ```
  </Accordion>
  
  <Accordion title="File Storage (S3)">
    For production, use S3-compatible storage:
    
    ```bash
    MM_FILESETTINGS_DRIVERNAME=amazons3
    MM_FILESETTINGS_AMAZONS3BUCKET=mattermost-files
    MM_FILESETTINGS_AMAZONS3REGION=us-east-1
    MM_FILESETTINGS_AMAZONS3ACCESSKEYID=your-access-key
    MM_FILESETTINGS_AMAZONS3SECRETACCESSKEY=your-secret-key
    ```
    
    Or use MinIO (included in docker-compose for development).
  </Accordion>
  
  <Accordion title="Performance Monitoring">
    Enable Prometheus metrics:
    
    ```bash
    MM_METRICSSETTINGS_ENABLE=true
    MM_METRICSSETTINGS_LISTENADDRESS=:8067
    ```
    
    Metrics endpoint: `http://localhost:8067/metrics`
  </Accordion>
</AccordionGroup>

## Backup and Recovery

### Database Backup

```bash
# Automated daily backup
sudo crontab -e

# Add this line for daily 2 AM backup
0 2 * * * pg_dump -U mmuser mattermost > /backup/mattermost-$(date +\%Y-\%m-\%d).sql
```

### Full Backup Script

```bash
#!/bin/bash
BACKUP_DIR="/backup/mattermost-$(date +%Y-%m-%d)"
mkdir -p $BACKUP_DIR

# Backup database
pg_dump -U mmuser mattermost > $BACKUP_DIR/database.sql

# Backup config
cp /opt/mattermost/config/config.json $BACKUP_DIR/

# Backup data directory
tar -czf $BACKUP_DIR/data.tar.gz /opt/mattermost/data

echo "Backup completed: $BACKUP_DIR"
```

## Upgrading Mattermost

<Warning>
  **Always backup before upgrading!** Test upgrades in a staging environment first.
</Warning>

### Docker Upgrade

```bash
# Pull new image
docker compose pull mattermost

# Restart with new version
docker compose up -d
```

### Binary Upgrade

```bash
# Stop Mattermost
sudo systemctl stop mattermost

# Backup current installation
sudo cp -ra /opt/mattermost /opt/mattermost-backup-$(date +%Y%m%d)

# Download new version
wget https://latest.mattermost.com/mattermost-enterprise-linux
tar -xvzf mattermost*.gz --transform='s,^[^/]\+,\0-upgrade,'

# Copy new files
sudo cp -an mattermost-upgrade/. /opt/mattermost/

# Restore permissions
sudo chown -R mattermost:mattermost /opt/mattermost

# Start Mattermost
sudo systemctl start mattermost
```

## Monitoring and Troubleshooting

### Health Checks

```bash
# Using mmctl (local mode)
/opt/mattermost/bin/mmctl --local system status

# HTTP health check
curl http://localhost:8065/api/v4/system/ping
```

### Log Locations

- **Docker**: `docker compose logs -f mattermost`
- **Systemd**: `sudo journalctl -u mattermost -f`
- **File**: `/opt/mattermost/logs/mattermost.log`

### Common Issues

<AccordionGroup>
  <Accordion title="Database Connection Failed">
    **Check:**
    - PostgreSQL is running: `sudo systemctl status postgresql`
    - Connection string format is correct
    - Database user has proper permissions
    - Firewall allows PostgreSQL port (5432)
    
    **Test connection:**
    ```bash
    psql "postgres://mmuser:password@localhost:5432/mattermost"
    ```
  </Accordion>
  
  <Accordion title="Port Already in Use">
    **Find what's using port 8065:**
    ```bash
    sudo lsof -i :8065
    ```
    
    **Change Mattermost port:**
    Set `MM_SERVICESETTINGS_LISTENADDRESS=:8066` or edit config.json
  </Accordion>
  
  <Accordion title="High Memory Usage">
    **Check:**
    - Number of concurrent users
    - Database connection pool size
    - File caching settings
    
    **Optimize:**
    ```bash
    MM_SQLSETTINGS_MAXOPENCONNS=100
    MM_FILESETTINGS_ENABLEFILECACHING=false
    ```
  </Accordion>
</AccordionGroup>
