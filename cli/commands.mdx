## Overview

This reference covers all mmctl commands, their options, and usage examples. Commands are organized by category for easy navigation.

## Global Flags

These flags work with all commands:

| Flag | Description |
|------|-------------|
| `--config` | Path to configuration file (default: `~/.mmctl/config`) |
| `--json` | Output in JSON format |
| `--local` | Use local mode (Unix socket) |
| `--strict` | Enforce version matching between mmctl and server |
| `--suppress-warnings` | Disable warning messages |
| `--insecure-sha1-intermediate` | Allow SHA-1 certificates |
| `--insecure-tls-version` | Allow TLS 1.0 and 1.1 |
| `--disable-pager` | Disable paged output |
| `--quiet` | Suppress output |

## Authentication Commands

<Accordion title="mmctl auth login">
  Login to a Mattermost instance and store credentials.

  **Syntax:**
  ```bash
  mmctl auth login [instance-url] [flags]
  ```

  **Flags:**
  - `--name, -n` - Name for the credentials
  - `--username, -u` - Username for authentication
  - `--password-file, -f` - File containing password
  - `--access-token-file, -t` - File containing access token
  - `--mfa-token, -m` - MFA token (if enabled)
  - `--no-activate` - Don't activate credentials after login

  **Examples:**
  ```bash
  # Interactive login
  mmctl auth login https://mattermost.example.com

  # Login with username and password file
  mmctl auth login https://mattermost.example.com \
    --name prod --username admin --password-file pass.txt

  # Login with MFA
  mmctl auth login https://mattermost.example.com \
    --name prod --username admin --password-file pass.txt --mfa-token 123456

  # Login with access token
  mmctl auth login https://mattermost.example.com \
    --name prod --access-token-file token.txt
  ```
</Accordion>

<Accordion title="mmctl auth current">

## User Commands

<Accordion title="mmctl user create">
  Create a new user.

  **Syntax:**
  ```bash
  mmctl user create [flags]
  ```

  **Flags:**
  - `--email` (required) - Email address
  - `--username` (required) - Username
  - `--password` (required) - Password
  - `--firstname` - First name
  - `--lastname` - Last name
  - `--nickname` - Nickname
  - `--locale` - Locale (e.g., en, fr)
  - `--system-admin` - Create as system administrator
  - `--guest` - Create as guest user
  - `--email-verified` - Mark email as verified
  - `--disable-welcome-email` - Don't send welcome email

  **Examples:**
  ```bash
  # Basic user
  mmctl user create --email user@example.com --username john --password Pass123!

  # User with additional details
  mmctl user create \
    --email user@example.com \
    --username john \
    --password Pass123! \
    --firstname John \
    --lastname Doe \
    --nickname JD

  # System administrator
  mmctl user create \
    --email admin@example.com \
    --username admin \
    --password AdminPass123! \
    --system-admin

  # Guest user
  mmctl user create \
    --email guest@example.com \
    --username guest1 \
    --password GuestPass123! \
    --guest
  ```
</Accordion>

<Accordion title="mmctl user list">
  List all users.

  **Syntax:**
  ```bash
  mmctl user list [flags]
  ```

  **Flags:**
  - `--page` - Page number (default: 0)
  - `--per-page` - Users per page (default: 200)
  - `--all` - Fetch all users
  - `--team` - Filter by team name or ID
  - `--inactive` - Show only inactive users
  - `--role` - Filter by role

  **Examples:**
  ```bash
  # List all users
  mmctl user list --all

  # List users in a specific team
  mmctl user list --team engineering

  # List inactive users
  mmctl user list --inactive

  # Paginated results
  mmctl user list --page 0 --per-page 50
  ```
</Accordion>

<Accordion title="mmctl user change-password">
  Change a user's password.

  **Syntax:**
  ```bash
  mmctl user change-password <user> [flags]
  ```

  **Flags:**
  - `--current, -c` - Current password (for own password)
  - `--password, -p` - New password
  - `--hashed` - Password is already hashed

  **Examples:**
  ```bash

  mmctl user change-password john.doe --password NewPass123!

  # Change your own password
  mmctl user change-password myusername \
    --current OldPass123! \
    --password NewPass123!

  # Interactive mode
  mmctl user change-password john.doe
  ```
</Accordion>

<Accordion title="mmctl user convert">
  Convert users to bots or bots to users.

  **Syntax:**
  ```bash
  mmctl user convert [users...] [flags]
  ```

  **Flags:**
  - `--bot` - Convert users to bots
  - `--user` - Convert bot to user
  - `--password` - Password for new user account
  - `--email` - Email for new user account
  - `--username` - Username for new user account

  **Examples:**
  ```bash
  # Convert user to bot
  mmctl user convert user@example.com --bot

  # Convert bot to user
  mmctl user convert botusername \
    --user \
    --email newuser@example.com \
    --password Pass123!
  ```
</Accordion>

## Post Commands

<Accordion title="mmctl post create">
  Create a post in a channel.

  **Syntax:**
  ```bash
  mmctl post create [channel] [flags]
  ```

  **Flags:**
  - `--message, -m` (required) - Post message
  - `--reply-to, -r` - Post ID to reply to

  **Examples:**
  ```bash
  # Create post
  mmctl post create engineering:general --message "Hello team!"

  # Reply to post
  mmctl post create engineering:general \
    --message "Great idea!" \
    --reply-to postid123
  ```
</Accordion>

<Accordion title="mmctl post list">
  List posts in a channel.

  **Syntax:**
  ```bash
  mmctl post list [channel] [flags]
  ```

  **Flags:**
  - `--number, -n` - Number of posts (default: 20)
  - `--show-ids, -i` - Show post IDs
  - `--follow, -f` - Follow channel for new posts
  - `--since, -s` - Show posts since time (ISO 8601)

  **Examples:**
  ```bash
  # List recent posts
  mmctl post list engineering:general

  # List 50 posts with IDs
  mmctl post list engineering:general --number 50 --show-ids

  # Follow channel for new posts
  mmctl post list engineering:general --follow

  # Posts since specific time
  mmctl post list engineering:general --since 2024-01-01T00:00:00Z
  ```
</Accordion>

<Accordion title="mmctl post delete">
  Delete posts.

  **Syntax:**
  ```bash
  mmctl post delete [posts...] [flags]
  ```

  **Flags:**
  - `--confirm` - Skip confirmation
  - `--permanent` - Permanently delete from database

  **Examples:**
  ```bash
  # Soft delete (mark as deleted)
  mmctl post delete postid123

  # Permanent delete
  mmctl post delete postid123 --permanent --confirm

  # Delete multiple posts
  mmctl post delete post1 post2 post3 --permanent
  ```
</Accordion>

## Permissions Commands

<Note>
  Permissions commands require Enterprise Edition.
</Note>

<Accordion title="mmctl permissions add">
  Add permissions to a role.

  **Syntax:**
  ```bash
  mmctl permissions add <role> <permission...>
  ```

  **Examples:**
  ```bash
  # Add single permission
  mmctl permissions add system_user list_open_teams

  # Add multiple permissions
  mmctl permissions add system_manager \
    sysconsole_read_user_management_channels \
    sysconsole_write_user_management_channels
  ```
</Accordion>

<Accordion title="mmctl permissions remove">
  Remove permissions from a role.

  **Syntax:**
  ```bash
  mmctl permissions remove <role> <permission...>
  ```

  **Example:**
  ```bash
  mmctl permissions remove system_user list_open_teams
  ```
</Accordion>

<Accordion title="mmctl permissions reset">
  Reset role to default permissions.

  **Syntax:**
  ```bash
  mmctl permissions reset <role>
  ```

  **Example:**
  ```bash
  mmctl permissions reset system_read_only_admin
  ```
</Accordion>

## Completion Command

<Accordion title="mmctl completion">
  Generate shell completion scripts.

  **Syntax:**
  ```bash
  mmctl completion [bash|zsh|fish|powershell]
  ```

  **Examples:**
  ```bash
  # Bash
  mmctl completion bash > /etc/bash_completion.d/mmctl

  # Zsh
  mmctl completion zsh > "${fpath[1]}/_mmctl"

  # Fish
  mmctl completion fish > ~/.config/fish/completions/mmctl.fish

  # PowerShell
  mmctl completion powershell > mmctl.ps1
  ```
</Accordion>
