## What is mmctl?

`mmctl` is a remote command-line interface (CLI) tool for managing your Mattermost server. It provides a powerful way to administer Mattermost instances from the terminal, whether you're managing users, teams, channels, or server configuration.

mmctl communicates with the Mattermost server through the REST API, making it ideal for:

- Server administration and maintenance
- Automation and scripting
- Bulk operations on users, teams, and channels
- Configuration management
- Integration with CI/CD pipelines

## Why Use mmctl?

### Remote Management
Manage your Mattermost server from anywhere without needing SSH access to the server. mmctl uses the Mattermost API to execute commands remotely.

### Automation-Friendly
mmctl is designed for scripting and automation. JSON output format support makes it easy to integrate with other tools and scripts.

### Comprehensive Coverage
mmctl provides commands for virtually all administrative tasks:

- **User Management**: Create, modify, activate/deactivate users
- **Team Management**: Create, archive, restore teams
- **Channel Management**: Create, archive, modify channels
- **Configuration**: Get, set, and manage server configuration
- **Plugins**: Install, enable, disable, and manage plugins
- **System Operations**: Check status, manage licenses, control server state
- **Permissions**: Manage roles and permissions (Enterprise Edition)
- **Posts**: Create, list, and delete posts

### Local Mode
For direct server access, mmctl supports local mode using Unix sockets, providing fast and secure communication without network overhead.

## Architecture

mmctl operates in two modes:

### Remote Mode (Default)

In remote mode, mmctl communicates with the Mattermost server via HTTPS using the REST API. This requires:

- Network connectivity to the Mattermost server
- Valid authentication credentials
- API access enabled on the server

```bash
# Authenticate to a remote server
mmctl auth login https://mattermost.example.com --name production
```

### Local Mode

Local mode uses Unix domain sockets for direct communication with the Mattermost server process. This mode:

- Requires local server access
- Provides faster performance
- Bypasses authentication for most commands
- Ideal for server maintenance tasks

```bash
# Use local mode
mmctl --local user list
```

## Output Formats

mmctl supports multiple output formats:

### Plain Text (Default)
Human-readable output formatted for terminal display.

```bash
mmctl user list
```

### JSON Format
Machine-readable JSON output for scripting and automation.

```bash
mmctl user list --json
```

## Common Use Cases

### Batch User Operations
Create multiple users, reset passwords, or modify user attributes in bulk using shell scripts.

### Infrastructure as Code
Manage server configuration as code by storing configuration files in version control.

### Monitoring and Alerting
Integrate mmctl commands into monitoring systems to check server health and status.

### Migration Tasks
Migrate users between authentication providers or move channels between teams.

### Backup Automation
Schedule regular configuration exports and data backups.
