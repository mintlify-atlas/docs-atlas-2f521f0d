Mattermost provides enterprise-grade security features including encryption, authentication providers, compliance controls, and security hardening options.

## Encryption

### Encryption at Rest

Mattermost supports encryption for data stored in the database and file storage.

**Database Encryption:**

```bash
# PostgreSQL transparent data encryption (TDE)
# Enable at the database level
ALTER DATABASE mattermost SET default_tablespace = encrypted_tablespace;

# Or use full disk encryption at OS level
# LUKS (Linux), BitLocker (Windows), FileVault (macOS)
```

**File Storage Encryption:**

```json
{
  "FileSettings": {
    "AmazonS3SSE": true,
    "AmazonS3Trace": false
  }
}
```

### Encryption in Transit

**TLS Configuration:**

```json
{
  "ServiceSettings": {
    "ConnectionSecurity": "TLS",
    "TLSCertFile": "/path/to/cert.pem",
    "TLSKeyFile": "/path/to/key.pem",
    "TLSMinVer": "1.2",
    "TLSStrictTransport": true,
    "TLSStrictTransportMaxAge": 63072000,
    "UseLetsEncrypt": false,
    "Forward80To443": true,
    "TLSOverwriteCiphers": [
      "TLS_ECDHE_RSA_WITH_AES_128_GCM_SHA256",
      "TLS_ECDHE_RSA_WITH_AES_256_GCM_SHA384",
      "TLS_ECDHE_ECDSA_WITH_AES_128_GCM_SHA256",
      "TLS_ECDHE_ECDSA_WITH_AES_256_GCM_SHA384"
    ]
  }
}
```

**Supported TLS Versions:**
- TLS 1.2 (recommended minimum)
- TLS 1.3 (recommended)

**Recommended Cipher Suites:**
- `TLS_ECDHE_RSA_WITH_AES_128_GCM_SHA256`
- `TLS_ECDHE_RSA_WITH_AES_256_GCM_SHA384`
- `TLS_ECDHE_ECDSA_WITH_AES_128_GCM_SHA256`
- `TLS_ECDHE_ECDSA_WITH_AES_256_GCM_SHA384`
- `TLS_ECDHE_RSA_WITH_CHACHA20_POLY1305`
- `TLS_ECDHE_ECDSA_WITH_CHACHA20_POLY1305`

> **Warning:** Never commit TLS certificates or private keys to version control. Use secure secret management solutions.

### Database Connection Security

```bash
# PostgreSQL with SSL
export MM_SQLSETTINGS_DATASOURCE="postgres://mmuser:password@localhost/mattermost?sslmode=verify-full&sslrootcert=/path/to/ca.crt&sslcert=/path/to/client.crt&sslkey=/path/to/client.key"
```

**SSL Modes:**
- `disable` - No SSL (not recommended for production)
- `require` - SSL required, but no verification
- `verify-ca` - Verify server certificate against CA
- `verify-full` - Verify server certificate and hostname (recommended)

```json
    "ScopingIDPName": "",
```

### Setting Up SAML

**Step 1: Generate Service Provider Metadata**

```bash
# Download metadata from Mattermost
curl https://mattermost.example.com/api/v4/saml/metadata > sp-metadata.xml
```

**Step 2: Upload Certificates**

```bash
# Upload IdP certificate
mmctl saml certificate upload-idp --cert idp.crt

# Upload service provider certificates
mmctl saml certificate upload-public --cert public.crt
mmctl saml certificate upload-private --cert private.key

# Verify certificates
mmctl saml certificate status
```

**Step 3: Configure Identity Provider**

Provide to your IdP administrator:
- Entity ID: `https://mattermost.example.com`
- ACS URL: `https://mattermost.example.com/login/sso/saml`
- Service Provider Metadata: `sp-metadata.xml`

**Step 4: Test SAML Login**

```bash
# Navigate to login page
https://mattermost.example.com/login/sso/saml
```

### SAML Attribute Mapping

| Mattermost Field | SAML Attribute | Required |
|------------------|----------------|----------|
| User ID | `IdAttribute` | Yes |
| Email | `EmailAttribute` | Yes |
| Username | `UsernameAttribute` | Yes |
| First Name | `FirstNameAttribute` | No |
| Last Name | `LastNameAttribute` | No |
| Guest Status | `GuestAttribute` | No |
| Admin Status | `AdminAttribute` | No |

## Multi-Factor Authentication (MFA)

### Enable MFA

```json
{
  "ServiceSettings": {
    "EnableMultifactorAuthentication": true,
    "EnforceMultifactorAuthentication": false
  }
}
```

**Enforce MFA for All Users:**

```bash
mmctl config set ServiceSettings.EnforceMultifactorAuthentication true
```

> **Note:** Users must set up MFA using TOTP apps like Google Authenticator, Authy, or 1Password.

### MFA Recovery

```bash
# If user loses MFA device, admin can reset:
mmctl user reset-mfa john.doe
```

## Password Security

### Password Policies

```json
{
  "PasswordSettings": {
    "MinimumLength": 10,
    "Lowercase": true,
    "Number": true,
    "Uppercase": true,
    "Symbol": true
  },
  "ServiceSettings": {
    "MaximumLoginAttempts": 10
  }
}
```

**Password Requirements:**
- Minimum length: 5-72 characters (recommended: 10+)
- Lowercase letters (optional)
- Uppercase letters (optional)
- Numbers (optional)
- Symbols (optional)

### Password Hashing

Mattermost uses bcrypt for password hashing:
- Algorithm: bcrypt
- Work factor: 10 (default)
- Automatic migration to latest hashing method

## Security Best Practices

> **Warning: Critical Security Practices**
>
> 1. **Always use TLS in production** - Never serve Mattermost over HTTP
> 2. **Enable MFA for admins** - Require MFA for system administrators
> 3. **Regular security updates** - Keep Mattermost updated to latest version
> 4. **Strong password policies** - Enforce minimum 10-character passwords
> 5. **Database encryption** - Enable encryption at rest for databases
> 6. **Secure secrets** - Never commit credentials to version control
> 7. **Audit logging** - Enable and monitor audit logs
> 8. **Network segmentation** - Isolate Mattermost in secure network zones
> 9. **Regular backups** - Maintain encrypted off-site backups
> 10. **Penetration testing** - Conduct regular security assessments

### Security Checklist

- [ ] TLS 1.2+ enabled with strong ciphers
- [ ] Database connections use SSL/TLS
- [ ] MFA enabled and enforced for admins
- [ ] Strong password policy configured
- [ ] Session timeouts configured
- [ ] Rate limiting enabled
- [ ] CSRF protection enabled
- [ ] Audit logging enabled
- [ ] Data retention policies configured
- [ ] Regular backups automated
- [ ] Security updates applied monthly
- [ ] Plugin signatures required
- [ ] IP allowlisting configured (if required)
- [ ] Compliance exports scheduled

## Security Monitoring

### Security Events to Monitor

```bash
# Failed login attempts
grep "invalid password" logs/mattermost.log

# Session anomalies
grep "session" logs/mattermost.log | grep -i "error\|fail"

# Plugin installations
grep "plugin" logs/mattermost.log

# Configuration changes
grep "config" logs/mattermost.log
```

### Integration with SIEM

Export logs to your SIEM solution:

```json
{
  "LogSettings": {

    "SyslogServerAddress": "syslog.example.com:514",
    "SyslogProtocol": "tcp",
    "SyslogTag": "mattermost",
    "SyslogCert": "/path/to/cert.pem",

  }

```

## Incident Response

### Security Incident Procedures

1. **Detect**: Monitor logs and alerts
2. **Contain**: Revoke sessions, disable accounts
3. **Investigate**: Review audit logs, access patterns
4. **Remediate**: Apply patches, update configurations
5. **Document**: Record incident details and lessons learned

```bash
# Emergency procedures

# Revoke all sessions
mmctl user revoke-sessions --all

# Disable compromised user
mmctl user deactivate compromised.user

# Export audit logs
mmctl logs export --output incident-logs.json

# Reset user passwords
mmctl user reset-password username --password TempSecure123!
```
