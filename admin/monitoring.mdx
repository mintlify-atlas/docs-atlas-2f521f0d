Proactive monitoring is essential for maintaining Mattermost performance and availability. This guide covers metrics collection, logging, health checks, and observability best practices.

## Health Checks

Mattermost provides built-in health check endpoints for monitoring service availability.

### API Health Endpoints

<CodeGroup>
```bash Basic Ping
# Simple health check
curl http://localhost:8065/api/v4/system/ping
# Response: {"status":"OK"}

# Check response time
time curl -s http://localhost:8065/api/v4/system/ping
```

```bash Detailed Health Check
# Comprehensive health check (requires authentication)
curl -H "Authorization: Bearer YOUR_TOKEN" \
  http://localhost:8065/api/v4/system/ping?get_server_status=true

# Response includes:
# - Database connectivity
# - File storage status
# - Cluster health (if enabled)
# - License status
```

```bash Docker Health Check
# Docker container health check
docker inspect --format='{{json .State.Health}}' mattermost-app | jq

# Using docker-compose
docker compose ps mattermost
```
</CodeGroup>

### Health Check Configuration

Docker Compose health check example:

```yaml
services:
  mattermost:
    image: mattermost/mattermost-team-edition:latest
    healthcheck:
      test: ["CMD", "curl", "-f", "http://localhost:8065/api/v4/system/ping"]
      interval: 30s
      timeout: 10s
      retries: 3
      start_period: 90s
```

Kubernetes probes:

```yaml
livenessProbe:
  httpGet:
    path: /api/v4/system/ping
    port: 8065
  initialDelaySeconds: 90
  periodSeconds: 30
  timeoutSeconds: 10
  failureThreshold: 3

readinessProbe:
  httpGet:
    path: /api/v4/system/ping
    port: 8065
  initialDelaySeconds: 30
  periodSeconds: 10
  timeoutSeconds: 5
  failureThreshold: 3
```

## Prometheus Metrics

Mattermost exposes Prometheus-compatible metrics on a dedicated port.

### Enable Metrics

<Steps>
  <Step title="Configure Metrics Endpoint">
    ```json
    {
      "MetricsSettings": {
        "Enable": true,
        "ListenAddress": ":8067",
        "BlockProfileRate": 0,
        "EnableClientMetrics": true,
        "EnableNotificationMetrics": true
      }
    }
    ```
    
    Or using environment variables:
    ```bash
    export MM_METRICSSETTINGS_ENABLE=true
    export MM_METRICSSETTINGS_LISTENADDRESS=":8067"
    ```
  </Step>
  
  <Step title="Verify Metrics Endpoint">
    ```bash
    # Check metrics are exposed
    curl http://localhost:8067/metrics
    
    # Count available metrics
    curl -s http://localhost:8067/metrics | grep "^mattermost_" | wc -l
    ```
  </Step>
  
  <Step title="Configure Prometheus">
    Create or update `prometheus.yml`:
    
    ```yaml
    global:
      scrape_interval: 15s
      evaluation_interval: 15s
    
    scrape_configs:
      - job_name: 'mattermost'
        static_configs:
          - targets: ['localhost:8067']
            labels:
              instance: 'mattermost-prod'
              environment: 'production'
        
        # For Docker environment
        # - targets: ['host.docker.internal:8067']
        
        # For Kubernetes
        # kubernetes_sd_configs:
        #   - role: pod
        #     namespaces:
        #       names:
        #         - mattermost
    ```
  </Step>
</Steps>

## Prometheus Setup

### Docker Compose with Prometheus

```yaml
services:
  prometheus:
    image: prom/prometheus:latest
    container_name: prometheus
    ports:
      - "9090:9090"
    volumes:
      - ./prometheus.yml:/etc/prometheus/prometheus.yml:ro
      - prometheus-data:/prometheus
    command:
      - '--config.file=/etc/prometheus/prometheus.yml'
      - '--storage.tsdb.path=/prometheus'
      - '--storage.tsdb.retention.time=30d'
    networks:
      - mattermost
    extra_hosts:
      - "host.docker.internal:host-gateway"

  grafana:
    image: grafana/grafana:latest
    container_name: grafana
    ports:
      - "3000:3000"
    volumes:
      - grafana-data:/var/lib/grafana
      - ./grafana/provisioning:/etc/grafana/provisioning:ro
    environment:
      - GF_SECURITY_ADMIN_PASSWORD=admin
      - GF_INSTALL_PLUGINS=grafana-piechart-panel
    networks:
      - mattermost
    depends_on:
      - prometheus

volumes:
  prometheus-data:
  grafana-data:
```

### Kubernetes Prometheus Operator

```yaml
apiVersion: monitoring.coreos.com/v1
kind: ServiceMonitor
metadata:
  name: mattermost

  labels:
    app: mattermost

  selector:
    matchLabels:
      app: mattermost

  - port: metrics
    interval: 30s
    path: /metrics
```

## Grafana Dashboards

### Configure Grafana Data Source

<Steps>
  <Step title="Access Grafana">
    Navigate to `http://localhost:3000`
    
    Default credentials:
    - Username: `admin`
    - Password: `admin`
  </Step>
  
  <Step title="Add Prometheus Data Source">
    1. Go to Configuration > Data Sources
    2. Click "Add data source"
    3. Select "Prometheus"
    4. Set URL: `http://prometheus:9090`
    5. Click "Save & Test"
  </Step>
  
  <Step title="Import Mattermost Dashboard">
    1. Go to Dashboards > Import
    2. Enter dashboard ID or upload JSON
    3. Select Prometheus data source
    4. Click "Import"
  </Step>
</Steps>

## Logging

Mattermost supports multiple logging outputs and formats.

### Log Configuration

```json
{
  "LogSettings": {
    "EnableConsole": true,
    "ConsoleLevel": "INFO",
    "ConsoleJson": true,
    "EnableColor": false,
    "EnableFile": true,
    "FileLevel": "INFO",
    "FileJson": true,
    "FileLocation": "/mattermost/logs/mattermost.log",
    "EnableWebhookDebugging": true,
    "EnableDiagnostics": true,
    "VerboseDiagnostics": false,
    "MaxFieldSize": 2048
  }
}
```

**Log Levels:**
- `DEBUG` - Detailed information for debugging
- `INFO` - General informational messages (recommended for production)
- `WARN` - Warning messages
- `ERROR` - Error messages
- `FATAL` - Critical errors causing shutdown

### Structured Logging

Enable JSON logging for better parsing:

```bash
export MM_LOGSETTINGS_CONSOLEJSON=true
export MM_LOGSETTINGS_FILEJSON=true
```

Sample JSON log entry:
```json
{
  "timestamp": "2024-03-15T14:30:45.123Z",
  "level": "info",
  "msg": "HTTP request",
  "caller": "web/handlers.go:123",
  "method": "POST",
  "path": "/api/v4/posts",
  "status": 201,
  "duration": 45.3,
  "user_id": "abc123"
}
```

**Promtail Configuration:**

```yaml

  http_listen_port: 9080
  grpc_listen_port: 0

      - timestamp:
```

## Performance Monitoring

### Application Performance

**Key Performance Indicators (KPIs):**

```promql
# Average response time
rate(mattermost_http_request_duration_seconds_sum[5m])
/ 
rate(mattermost_http_request_duration_seconds_count[5m])

# Request throughput (requests per second)
rate(mattermost_http_requests_total[5m])

# Error rate percentage
rate(mattermost_http_errors_total[5m]) 
/ 
rate(mattermost_http_requests_total[5m]) * 100

# Database query time (p99)
histogram_quantile(0.99,
  rate(mattermost_db_store_duration_seconds_bucket[5m])
)
```

### Resource Monitoring

**CPU Usage:**
```promql
rate(process_cpu_seconds_total{job="mattermost"}[5m]) * 100
```

**Memory Usage:**
```promql
process_resident_memory_bytes{job="mattermost"} / 1024 / 1024
```

**Disk I/O:**
```promql
rate(process_disk_read_bytes_total{job="mattermost"}[5m])
rate(process_disk_written_bytes_total{job="mattermost"}[5m])
```

**Network Traffic:**
```promql
rate(process_network_receive_bytes_total{job="mattermost"}[5m])
rate(process_network_transmit_bytes_total{job="mattermost"}[5m])
```

## Alerting

### Prometheus Alerts

Create `alerts.yml`:

```yaml
groups:
  - name: mattermost
    interval: 30s
    rules:
      - alert: MattermostDown
        expr: up{job="mattermost"} == 0
        for: 1m
        labels:
          severity: critical
        annotations:
          summary: "Mattermost instance is down"
          description: "Mattermost has been down for more than 1 minute"
      
      - alert: HighErrorRate
        expr: |
          rate(mattermost_http_errors_total[5m]) 
          / 
          rate(mattermost_http_requests_total[5m]) > 0.05
        for: 5m
        labels:
          severity: warning
        annotations:
          summary: "High error rate detected"
          description: "Error rate is {{ $value | humanizePercentage }}"
      
      - alert: HighResponseTime
        expr: |
          histogram_quantile(0.95,
            rate(mattermost_http_request_duration_seconds_bucket[5m])
          ) > 2
        for: 5m
        labels:
          severity: warning
        annotations:
          summary: "High response time"
          description: "95th percentile response time is {{ $value }}s"
      
      - alert: DatabaseConnectionIssue
        expr: mattermost_db_store_duration_seconds > 10
        for: 2m
        labels:
          severity: critical
        annotations:
          summary: "Database connection issues"
          description: "Database queries taking too long"
      
      - alert: HighMemoryUsage
        expr: |
          process_resident_memory_bytes{job="mattermost"} 
          / 1024 / 1024 / 1024 > 8
        for: 10m
        labels:
          severity: warning
        annotations:
          summary: "High memory usage"
          description: "Memory usage is {{ $value }}GB"
      
      - alert: TooManyGoroutines
        expr: go_goroutines{job="mattermost"} > 10000
        for: 5m
        labels:
          severity: warning
        annotations:
          summary: "Too many goroutines"
          description: "Goroutine count is {{ $value }}"
```

### Alertmanager Configuration

```yaml
route:
  receiver: 'default'
  group_by: ['alertname', 'severity']
  group_wait: 30s
  group_interval: 5m
  repeat_interval: 4h
  
  routes:
    - match:
        severity: critical
      receiver: 'pagerduty'
    - match:
        severity: warning
      receiver: 'slack'

receivers:
  - name: 'default'
    email_configs:
      - to: 'ops@example.com'
        from: 'alertmanager@example.com'
        smarthost: 'smtp.example.com:587'
  
  - name: 'slack'
    slack_configs:
      - api_url: 'https://hooks.slack.com/services/YOUR/WEBHOOK/URL'
        channel: '#alerts'
        title: 'Mattermost Alert'
        text: '{{ range .Alerts }}{{ .Annotations.summary }}\n{{ end }}'
  
  - name: 'pagerduty'
    pagerduty_configs:
      - service_key: 'YOUR_PAGERDUTY_KEY'
```

## Troubleshooting

### Metrics Not Available

```bash
# Check metrics endpoint is enabled
grep -i "metrics" /opt/mattermost/config/config.json

# Test metrics endpoint
curl -v http://localhost:8067/metrics

# Check firewall
sudo iptables -L -n | grep 8067
```

### High Memory Usage

```bash
# Check memory usage
free -h
docker stats mattermost-app

# Review Go heap profile
curl http://localhost:8067/debug/pprof/heap > heap.prof
go tool pprof heap.prof
```

### Slow Queries

```sql
-- PostgreSQL slow query log
ALTER DATABASE mattermost SET log_min_duration_statement = 1000;

-- Check slow queries
SELECT * FROM pg_stat_statements 
ORDER BY mean_exec_time DESC 
LIMIT 10;
```
